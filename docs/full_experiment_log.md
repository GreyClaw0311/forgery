## 第一阶段：图像级篡改检测 (03-11 ~ 03-13)

### 1.1 目标
判断一张图片是否被篡改（二分类问题）

### 1.2 数据集
- **Easy**: 20张篡改图片
- **Difficult**: 17张篡改图片  
- **Good**: 正常图片（测试误报）

### 1.3 特征工程

实现了24个图像篡改检测特征：

| 类别 | 特征 | 说明 |
|------|------|------|
| DCT | dct, dct_low, dct_high | JPEG压缩痕迹 |
| ELA | ela_mean, ela_std | 错误级别分析 |
| Noise | noise, noise_var | 噪声一致性 |
| Color | color, saturation | 颜色分布异常 |
| Edge | edge | 边缘特征 |
| CFA | cfa | 颜色过滤阵列 |
| JPEG | jpeg_block, jpeg_ghost | JPEG块效应 |
| Splicing | splicing | 拼接痕迹 |
| Resampling | resampling | 重采样检测 |
| Contrast | contrast | 对比度异常 |

**Top 10 有效特征**:
1. fft (频域分析)
2. resampling (重采样)
3. color (颜色)
4. jpeg_block (JPEG块)
5. splicing (拼接)
6. cfa (颜色过滤)
7. contrast (对比度)
8. edge (边缘)
9. jpeg_ghost (JPEG幽灵)
10. saturation (饱和度)

### 1.4 模型训练

**算法**: Gradient Boosting

**交叉验证结果**:
- CV F1 Score: **0.9742**

**测试集结果**:
- F1 Score: **0.9302**

### 1.5 数据不平衡处理

**问题**: 正常样本仅占4.9%，导致误报率84.21%

**解决方案**: 类别权重调整
- 正常样本权重: 10.14
- 篡改样本权重: 0.53

**结果**: 误报率降至36.84%

### 1.6 阈值优化

通过网格搜索确定最优阈值0.85

**权衡关系**:
- 阈值↑ → 误报率↓，漏报率↑
- 阈值↓ → 误报率↑，漏报率↓

## 第二阶段：像素级篡改分割 (03-14)

### 2.1 目标转变
从"判断图片是否篡改"转变为"精确定位篡改区域"

### 2.2 方法对比

| 方法 | Precision | Recall | F1 | 输出 |
|------|-----------|--------|------|------|
| 传统方法 (特征+阈值) | 14.7% | 56.8% | 0.186 | 概率图 |
| **像素级ML** ✅ | **91.2%** | **88.4%** | **0.898** | **精确Mask** |
| 提升 | +76.5% | +31.6% | **+382%** | |

### 2.3 核心设计

#### 滑动窗口策略
- 窗口大小: 32×32像素
- 滑动步长: 8像素
- 每个窗口提取特征 → 预测中心像素

#### 57维特征向量

| 特征类别 | 特征数 | 说明 |
|----------|--------|------|
| DCT | 5 | 频域特征, JPEG压缩痕迹 |
| ELA | 4 | Error Level Analysis |
| Noise | 4 | 噪声一致性 |
| Edge | 4 | 边缘特征 |
| LBP | 32 | 纹理特征 |
| Color | 8 | 颜色/HSV特征 |
| **总计** | **57** | |

#### 特征重要性 Top 5

| 排名 | 特征 | 重要性 | 说明 |
|------|------|--------|------|
| 1 | **Noise_p95** | 7.30% | 噪声95分位数 |
| 2 | Edge_mag_max | 6.13% | 边缘强度最大值 |
| 3 | Edge_mag_std | 5.89% | 边缘强度标准差 |
| 4 | Edge_mag_mean | 5.46% | 边缘强度均值 |
| 5 | Edge_density | 5.32% | 边缘密度 |

### 2.4 模型配置

- **算法**: Random Forest
- **树数量**: 100
- **最大深度**: 20
- **类别权重**: balanced (处理数据不平衡)

### 2.5 训练结果

**数据集**:
- 图片数: 37张 (Easy 20 + Difficult 17)
- 像素样本: 30,752
- 篡改像素: 8,426 (27.4%)
- 正常像素: 22,326 (72.6%)

**性能指标**:
- 训练准确率: 96.57%
- **验证准确率: 94.49%**
- **Precision: 91.19%**
- **Recall: 88.43%**
- **F1: 0.898** ✅

**最佳阈值**: 0.5

### 2.6 关键发现

1. **噪声特征最重要** - 篡改区域噪声分布异常
2. **边缘特征次之** - 篡改边缘往往不自然
3. **DCT/ELA仍然有效** - JPEG压缩痕迹是重要线索
4. **多特征融合有效** - 57维特征综合判断

## 第四阶段：优化实验 (03-15)

### 4.1 实验设计

**目标**: 利用全量数据提升F1到0.85以上

**策略**:
- 扩大训练集：1000张图片，500,000个像素样本
- 优化特征：从57维精简到35维关键特征
- 加快迭代：滑动步长从8增加到16
- 增强模型：树数量从100增加到200

### 4.2 实验结果

**数据集**:
- 图片数: 1,000张
- 像素样本: 500,000个
- 窗口: 32×32
- 步长: 16
- 特征维度: 35

**性能指标**:
- 训练准确率: -
- **验证准确率: 96.67%**
- **最佳阈值: 0.35**
- **F1: 0.5657** ❌

**目标未达成**: F1未达到0.85

### 4.3 分析

**可能原因**:
1. 数据量增加但质量参差不齐
2. 特征精简过度，丢失关键信息
3. 步长增大导致空间分辨率下降
4. 数据不平衡加剧（篡改像素比例下降）

**教训**:
- 小数据集实验成功不代表大数据集也能成功
- 需要更精细的数据清洗和平衡策略
- 特征工程需要针对大数据集重新调优

## 核心经验总结

### ✅ 成功经验

1. **像素级ML远优于传统阈值方法** - F1从0.186提升到0.898
2. **噪声和边缘特征最重要** - 篡改区域的噪声和边缘不自然
3. **多特征融合有效** - 57维特征综合判断
4. **Random Forest足够** - 不需要复杂深度学习
5. **数据不平衡处理关键** - 类别权重调整显著改善效果

### ❌ 失败教训

1. **小数据成功≠大数据成功** - 需要针对性调优
2. **特征精简需谨慎** - 过度精简会丢失关键信息
3. **步长影响大** - 空间分辨率下降导致性能下降
4. **数据质量>数量** - 盲目增加数据不一定有效

### 🎯 最佳实践

1. **保持简单**: 单特征 + 好阈值 + 后处理
2. **实验对比**: 控制变量，对比验证
3. **迭代优化**: 小步快跑，快速验证
4. **记录完整**: 实验过程、结果、分析都要记录
